## Structural Defense Preserved

The unit test's failing-old-regex assertion is intentionally retained even after REFACTOR consolidated the production call through `_strip_tag_output_prefix`:

```python
# Step 4: independently verify the OLD regex fails on this fixture,
# proving the test fixture captures the actual regression shape.
old_stripped = re.sub(r"^\[[^\]]+\]\s*", "", payload_raw)
assert old_stripped.startswith("<stdout>:"), ...
with pytest.raises(json.JSONDecodeError):
    json.loads(old_stripped)
```

This is a two-sided lock:
1. The production helper (`_strip_tag_output_prefix(payload_raw)`) produces valid JSON → `json.loads` succeeds.
2. The OLD broken pattern (`r"^\[[^\]]+\]\s*"`, inlined verbatim) produces invalid JSON → `json.loads` raises `JSONDecodeError`.

A future regression that reverts `_TAG_OUTPUT_PREFIX_RE` to the broken pattern would fail (1) — the helper would no longer produce valid JSON. The OLD-regex sanity check (2) is independent of the helper and proves the fixture genuinely captures the OpenMPI 4.x regression shape (not a synthetic test case that would pass with any regex).
